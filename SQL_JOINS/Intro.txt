## üü¢ 1. **INNER JOIN**

> Returns rows that **exist in both** tables.

```sql
SELECT e.name, d.dept_name
FROM employees e
INNER JOIN departments d
ON e.dept_id = d.dept_id;
```

**Result:**

| name  | dept_name |
| ----- | --------- |
| Alice | HR        |
| Bob   | IT        |

‚úÖ Only matching dept_id (10 & 20).
Charlie‚Äôs 30 and David‚Äôs NULL don‚Äôt exist in `departments`.

---

## üîµ 2. **LEFT JOIN** (a.k.a. LEFT OUTER JOIN)

> Returns **all rows from the left table**, and matching ones from the right.
> If no match ‚Üí `NULL` on the right side.

```sql
SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d
ON e.dept_id = d.dept_id;
```

**Result:**

| name    | dept_name |
| ------- | --------- |
| Alice   | HR        |
| Bob     | IT        |
| Charlie | NULL      |
| David   | NULL      |

‚úÖ Everyone from `employees` appears ‚Äî even if not in `departments`.

---

## üü† 3. **RIGHT JOIN** (a.k.a. RIGHT OUTER JOIN)

> Opposite of LEFT JOIN ‚Äî returns **all rows from the right table** and matching ones from the left.

```sql
SELECT e.name, d.dept_name
FROM employees e
RIGHT JOIN departments d
ON e.dept_id = d.dept_id;
```

**Result:**

| name  | dept_name |
| ----- | --------- |
| Alice | HR        |
| Bob   | IT        |
| NULL  | Finance   |

‚úÖ All departments appear ‚Äî even if no employee belongs (Finance has none).

---

## üî¥ 4. **CROSS JOIN**

> Returns **every possible combination** of rows between both tables.

```sql
SELECT e.name, d.dept_name
FROM employees e
CROSS JOIN departments d;
```

**Result:** (4 employees √ó 3 departments = 12 rows!)

| name    | dept_name |
| ------- | --------- |
| Alice   | HR        |
| Alice   | IT        |
| Alice   | Finance   |
| Bob     | HR        |
| Bob     | IT        |
| Bob     | Finance   |
| Charlie | HR        |
| ...     | ...       |

‚úÖ Used rarely ‚Äî mostly for testing or special calculations.

## üßç‚Äç‚ôÇÔ∏è 5. **SELF JOIN**

> A table joins **with itself** ‚Äî useful for hierarchical data (like managers and employees).

Example:
Let‚Äôs say `employees` has a `manager_id` column referencing another employee.

| emp_id | name    | manager_id |
| ------ | ------- | ---------- |
| 1      | Alice   | NULL       |
| 2      | Bob     | 1          |
| 3      | Charlie | 1          |
| 4      | David   | 2          |

```sql
SELECT e.name AS employee, m.name AS manager
FROM employees e
LEFT JOIN employees m
ON e.manager_id = m.emp_id;
```

**Result:**

| employee | manager |
| -------- | ------- |
| Alice    | NULL    |
| Bob      | Alice   |
| Charlie  | Alice   |
| David    | Bob     |

‚úÖ You can give the same table **aliases (e, m)** to treat it like two.

## üß© 6. **UNION**

> Combines results from **two SELECTs** and removes duplicates.

```sql
SELECT name FROM employees
UNION
SELECT dept_name FROM departments;
```

**Result:**

```
Alice
Bob
Charlie
David
HR
IT
Finance
```

‚úÖ Removes duplicates automatically.
‚ö†Ô∏è Each SELECT must return the **same number of columns** and compatible data types.

---

## üß± 7. **UNION ALL**

> Same as UNION, but **keeps duplicates**.

```sql
SELECT name FROM employees
UNION ALL
SELECT dept_name FROM departments;
```

**Result:**

```
Alice
Bob
Charlie
David
HR
IT
Finance
```

(If there were overlaps, they‚Äôd appear multiple times.)

‚úÖ Faster than `UNION` since it doesn‚Äôt remove duplicates.

## üéì Quick Summary

| Type           | What it Does                       | Keeps Unmatched? |
| -------------- | ---------------------------------- | ---------------- |
| **INNER JOIN** | Only matching rows                 | ‚ùå                |
| **LEFT JOIN**  | All from left + matches            | ‚úÖ Left side      |
| **RIGHT JOIN** | All from right + matches           | ‚úÖ Right side     |
| **CROSS JOIN** | All combinations                   | N/A              |
| **SELF JOIN**  | Join table to itself               | N/A              |
| **UNION**      | Combine results, remove duplicates | ‚ùå                |
| **UNION ALL**  | Combine results, keep duplicates   | ‚úÖ                |
